What is MCP?

The Model Context Protocol is an open standard that lets an AI assistant plug into external knowledge servers. Think of it as a USB port for AI: plug one in and your assistant gains whatever capabilities the server exposes. This particular server exposes Cardano developer knowledge — documentation, smart-contract references, SDK guides, standards — so your assistant can answer Cardano questions with citations to real upstream docs instead of making things up.

Where does the knowledge come from?

From a fixed, public allowlist of GitHub repositories — every single one listed in config/sources.yaml. The server never performs a live web search, never fetches arbitrary URLs, and never reads from sources the maintainer has not explicitly accepted. Every answer returned by the server carries a citation to the exact source file, so you can always verify where it came from.

Who runs it and how is it vetted?

This is an independent project run by Easy1Staking. It is not affiliated with any foundation, IOG, Intersect, or ecosystem body. Changes to the allowlist happen through public pull requests that are schema-validated by CI and manually reviewed against the acceptance criteria. No tokens, no dApps, no product endorsements, no financial advice.

Get Started

Connect your AI assistant to the Cardano knowledge base in seconds. Use the hosted instance or run your own.

Add to your Claude Desktop configuration (claude_desktop_config.json):

{
  "mcpServers": {
    "cardano": {
      "url": "https://mcp.easy1staking.com/mcp"
    }
  }
}

Add to your Claude Code MCP settings (.claude/settings.json or via /mcp command):

{
  "mcpServers": {
    "cardano": {
      "url": "https://mcp.easy1staking.com/mcp"
    }
  }
}

Or run directly in the CLI:

claude mcp add cardano --transport streamable-http --url https://mcp.easy1staking.com/mcp

Go to Settings → MCP and add a new server:

{
  "mcpServers": {
    "cardano": {
      "url": "https://mcp.easy1staking.com/mcp"
    }
  }
}

Add to your Windsurf MCP configuration:

{
  "mcpServers": {
    "cardano": {
      "serverUrl": "https://mcp.easy1staking.com/mcp"
    }
  }
}

Any MCP-compatible client can connect via the streamable HTTP transport:

Endpoint: https://mcp.easy1staking.com/mcp
Transport: Streamable HTTP (SSE)
Auth: None required (public instance)

For local/stdio mode:

{
  "mcpServers": {
    "cardano": {
      "command": "node",
      "args": ["/path/to/cardano-unified-mcp-server/dist/index.js", "--stdio"]
    }
  }
}

Self-Hosting

Docker

docker build -t cardano-mcp .
docker run -p 3000:3000 \
  -e EMBEDDINGS_API_KEY=sk-... \
  cardano-mcp

From Source

git clone https://github.com/easy1staking-com/cardano-unified-mcp-server.git
cd cardano-unified-mcp-server
npm install && npm run build

# Ingest docs (needs OpenAI key for semantic search)
EMBEDDINGS_API_KEY=sk-... npm run ingest

# Start server
npm start

Kubernetes / Helm

helm install cardano-mcp ./helm/cardano-mcp \
  --set secrets.embeddingsApiKey=sk-... \
  --set ingress.enabled=true \
  --set ingress.host=mcp.yourdomain.com

See the Helm chart for full configuration options including TLS, session affinity, and ingestion CronJobs.

Architecture

A single stateless Node.js process. MCP clients connect over stdio or streamable HTTP. All reads hit a local SQLite database that combines BM25 full-text search (via FTS5) and vector similarity (via sqlite-vec). No outbound network calls at query time — no live web search, no external API fetches.

Documentation is re-ingested weekly by a Kubernetes CronJob: config/sources.yaml → git clone → format-aware parse → chunk → embed → atomic upsert. The full component, query-flow, and ingestion-flow diagrams (in Mermaid) live in docs/architecture.md.

Search Modes

  • Hybrid (default) — BM25 full-text (40%) + vector similarity (60%)
  • Semantic — Embedding cosine similarity only
  • Keyword — Full-text search with Porter stemming (no API key needed)

Configuration

  • PORT — HTTP port (default: 3000)
  • EMBEDDINGS_API_KEY — OpenAI key for semantic search
  • MCP_API_KEY — Optional auth for HTTP mode
  • DB_PATH — SQLite database path

Contributing

This is a neutral knowledge server for people building on Cardano. Contributions welcome — especially new documentation sources. Please read ABOUT.md first for the project's scope, trust model, and acceptance criteria.

Add a Documentation Source

Edit config/sources.yaml and open a PR. See the schema reference for every field, and the acceptance criteria for what qualifies. Every PR is schema-validated by CI.

Found a bug, a broken source, or something misleading? Open an issue. GitHub issues are the only intake channel — no email, no Discord.
